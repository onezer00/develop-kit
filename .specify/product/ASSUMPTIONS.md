# PRODUCT ASSUMPTIONS

> Este documento registra todas as hipóteses do produto que ainda não foram validadas.
>
> Nenhuma hipótese deve ser tratada como uma decisão até que existam dados suficientes para comprová-la.
>
> O objetivo deste documento é orientar os experimentos do MVP.

# A-0001

## Data

2026-07-07

## Status

Testing

## Categoria

UX

## Prioridade

Critical

## Hipótese

Usuários preferem realizar login utilizando Google ao invés de criar uma conta manualmente.

## Justificativa

Reduz atrito no onboarding.

## Como validar

Comparar:

- Taxa de conclusão do cadastro.
- Tempo até primeiro uso.
- Conversão entre Login Google e Login tradicional.

## Métrica de sucesso

Pelo menos 70% dos novos usuários optarem pelo Login Google.

## Impacto caso seja verdadeira

Login Google passa a ser o fluxo principal.

## Impacto caso seja falsa

Melhorar o fluxo de cadastro tradicional.

# A-0002

## Data

2026-07-07

## Status

Testing

## Categoria

Product

## Prioridade

Critical

## Hipótese

Desenvolvedores utilizarão a plataforma várias vezes durante o mesmo dia.

## Justificativa

O produto foi concebido para ser um companion diário.

## Como validar

Medir:

- Sessões por usuário.
- Ferramentas utilizadas por sessão.
- Retorno diário.

## Métrica de sucesso

Mais de 3 sessões por usuário ativo por dia.

## Impacto caso seja verdadeira

Expandir funcionalidades voltadas para produtividade.

## Impacto caso seja falsa

Reposicionar o produto como ferramenta ocasional.
