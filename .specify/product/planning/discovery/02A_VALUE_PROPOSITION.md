# VALUE PROPOSITION

> Este documento define a proposta de valor oficial do produto.
>
> Ele responde à pergunta mais importante de toda a fase de Product Discovery:
>
> **"Por que alguém escolheria nossa plataforma em vez de continuar utilizando Google, ChatGPT, extensões do VSCode e dezenas de ferramentas isoladas?"**
>
> Todas as decisões futuras de Produto, UX, Branding, Marketing e Desenvolvimento deverão estar alinhadas a esta proposta de valor.

---

# Objetivo

Construir uma plataforma que permita aos desenvolvedores resolver tarefas recorrentes com o menor esforço cognitivo possível, reduzindo interrupções e permitindo que retornem rapidamente ao desenvolvimento.

Nosso objetivo não é oferecer mais ferramentas.

Nosso objetivo é permitir que o desenvolvedor permaneça em estado de fluxo ("Flow State") durante o maior tempo possível.

---

# Problema

O fluxo diário de um desenvolvedor é constantemente interrompido.

Durante um único dia de trabalho é comum alternar entre:

- IDE
- Navegador
- ChatGPT
- Google
- Stack Overflow
- Documentação oficial
- Conversores online
- JWT
- Base64
- JSON
- Regex
- UUID
- SQL Formatter
- Markdown Preview
- APIs
- Terminal

Cada troca de contexto possui um custo.

Mesmo interrupções de poucos segundos reduzem concentração e produtividade.

O problema não é a ausência de ferramentas.

O problema é a fragmentação delas.

---

# Nossa Visão

Acreditamos que pequenas interrupções representam um dos maiores desperdícios de produtividade durante o desenvolvimento de software.

Queremos eliminar esse atrito.

Nosso produto será um espaço único para resolução de tarefas recorrentes.

---

# Nossa Missão

Permitir que desenvolvedores resolvam tarefas auxiliares em poucos segundos e retornem imediatamente ao desenvolvimento.

---

# Nossa Promessa

Reduzir a carga cognitiva do desenvolvedor.

Não prometemos:

- escrever código automaticamente;
- substituir IDEs;
- substituir ChatGPT;
- substituir documentação.

Prometemos tornar essas ferramentas menos necessárias durante tarefas repetitivas.

# O que vendemos

Não vendemos ferramentas.

Não vendemos IA.

Não vendemos conversores.

Vendemos:

- produtividade;
- foco;
- continuidade;
- velocidade;
- simplicidade.

---

# Nossa Diferenciação

Os concorrentes organizam seus produtos por ferramentas.

Nós organizaremos por problemas e fluxos de trabalho.

Exemplo:

Authentication Workflow

- JWT
- Base64
- UUID
- Hash
- Secrets

Data Workflow

- JSON
- YAML
- XML
- CSV
- SQL

Networking Workflow

- URL Encode
- Headers
- HTTP Status
- cURL
- REST

O usuário pensa em problemas.

Não em ferramentas.

---

# O que NÃO seremos

Não seremos:

- um novo ChatGPT;
- um IDE;
- uma IDE online;
- um clone do DevToys;
- um clone do CyberChef;
- um catálogo de links.

Toda funcionalidade deverá reforçar esse posicionamento.

---

# Público-Alvo

Nosso foco inicial serão desenvolvedores que:

- alternam frequentemente entre ferramentas;
- resolvem muitas tarefas pequenas durante o dia;
- valorizam velocidade;
- trabalham diariamente com APIs, dados, autenticação, debugging ou infraestrutura.

O ICP oficial será definido em `03_IDEAL_CUSTOMER_PROFILE.md.md`.

---

# Nossa Hipótese Central

Desenvolvedores preferem uma experiência consistente e integrada do que dezenas de ferramentas isoladas espalhadas pela internet.

Esta hipótese será validada durante o MVP.

---

# Benefícios Esperados

Usuários deverão perceber:

- menor troca de contexto;
- maior velocidade para tarefas repetitivas;
- menor carga cognitiva;
- maior organização;
- experiência consistente;
- confiança;
- menor necessidade de pesquisar soluções simples.

---

# Pilares do Produto

Todo recurso deverá fortalecer pelo menos um destes pilares.

## 1. Velocidade

Resolver tarefas em poucos segundos.

---

## 2. Simplicidade

Poucos cliques.

Poucas decisões.

Pouca configuração.

---

## 3. Consistência

Interface uniforme.

Mesmo comportamento em toda a plataforma.

---

## 4. Privacidade

Processamento local sempre que possível.

Dados pertencem ao usuário.

---

## 5. Inteligência

IA será utilizada apenas quando reduzir esforço.

Nunca como diferencial de marketing.

---

# Proposta de Valor

## Elevator Pitch

Uma plataforma de produtividade para desenvolvedores que reduz a troca de contexto, centraliza tarefas recorrentes e permite que o usuário permaneça focado no desenvolvimento.

---

## One-liner

Everything developers need between writing code.

---

## Tagline (candidatas)

- Stay in Flow.
- Less Context Switching.
- Build Faster.
- Developer Productivity Platform.
- Focus on Code.
- Everything Between Commits.
- One Workspace. Zero Distractions.
- Stop Searching. Start Building.
- Your Daily Developer Workspace.

---

# Critérios de Validação

Esta proposta de valor será considerada validada quando:

- usuários retornarem frequentemente à plataforma;
- a maioria das sessões envolver mais de uma capability;
- houver redução perceptível na necessidade de acessar ferramentas externas;
- usuários demonstrarem intenção de substituir favoritos do navegador pela plataforma.
