# User Journey

> Este documento descreve a jornada inicial esperada para o usuário do MVP.

---

# Jornada principal

## 1. Descoberta

O usuário chega por:

- busca orgânica por uma capability específica;
- indicação em comunidade;
- link direto compartilhado;
- conteúdo técnico;
- favoritos do navegador.

## Necessidade nesse momento

Resolver uma tarefa imediata sem cadastro obrigatório.

## Requisitos de experiência

- Página rápida.
- Capability utilizável imediatamente.
- Proposta de valor clara sem bloquear a tarefa.
- Sem anúncios.

---

# 2. Primeiro uso

O usuário cola ou digita um input, executa a ação desejada e obtém resultado em poucos segundos.

## Exemplo

Decodificar um JWT, formatar JSON ou converter Base64.

## Requisitos de experiência

- Input evidente.
- Resultado instantâneo ou quase instantâneo.
- Copiar resultado com um clique.
- Indicação clara quando o processamento é local.
- Erros compreensíveis.

---

# 3. Exploração

Após concluir a primeira tarefa, o usuário percebe capabilities relacionadas ou workflows.

## Exemplo

Ao usar JWT Decoder, encontra Base64, Timestamp, UUID e Hash dentro do Authentication Workflow.

## Requisitos de experiência

- Sugestões contextuais.
- Navegação sem perder o input.
- Busca global sempre acessível.
- URLs amigáveis para retorno rápido.

---

# 4. Retorno

O usuário volta à plataforma em outro momento do dia para uma tarefa parecida.

## Gatilhos

- Favorito do navegador.
- Histórico.
- Busca orgânica.
- Memória da experiência anterior.
- URL direta.

## Requisitos de experiência

- Últimas capabilities usadas.
- Histórico local ou associado à conta quando autenticado.
- Favoritos.
- Preferências persistentes.

---

# 5. Conta opcional

O usuário cria conta quando percebe valor em continuidade.

## Motivações

- Sincronizar favoritos.
- Manter histórico.
- Personalizar workspace.
- Acessar recursos futuros.

## Requisitos de experiência

- Login Google como caminho principal.
- Cadastro tradicional como alternativa.
- Não bloquear capabilities básicas antes da validação do MVP.

---

# 6. Uso recorrente

O usuário passa a tratar a plataforma como workspace auxiliar diário.

## Sinais de sucesso

- Usa busca global com frequência.
- Abre mais de uma capability por sessão.
- Retorna várias vezes por semana.
- Substitui ferramentas isoladas e favoritos antigos.

---

# Journey Map

| Etapa | Objetivo do usuário | Risco | Resposta do produto |
| --- | --- | --- | --- |
| Descoberta | Resolver algo agora | Página parecer marketing | Capability utilizável no primeiro viewport |
| Primeiro uso | Obter resultado rápido | Fricção ou cadastro | Uso imediato e sem anúncios |
| Exploração | Encontrar tarefa relacionada | Produto parecer catálogo | Workflows contextuais |
| Retorno | Repetir tarefa | Esquecer o produto | Histórico, favoritos e URLs claras |
| Conta | Ganhar continuidade | Medo de cadastro inútil | Login simples e benefício explícito |
| Recorrência | Workspace diário | Baixo hábito | Busca global, atalhos e personalização |

---

# Princípio de jornada

O usuário deve conseguir obter valor antes de entender o produto inteiro.
