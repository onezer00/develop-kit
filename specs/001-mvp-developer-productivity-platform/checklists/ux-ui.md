# UX/UI Checklist: MVP Developer Productivity Platform

**Purpose**: Validate that the user experience and visual design are modern, clear, consistent, accessible, and suitable for repeated developer use.
**Created**: 2026-07-09
**Feature**: [spec.md](../spec.md)

## Experience Quality

- [ ] CHK001 The landing page shows a clear hero section, product value, primary Google Auth action, and a visual identity that feels modern rather than generic.
- [ ] CHK002 Authentication, onboarding, workspace, and capability screens use the same visual language, typography, spacing, interaction states, and navigation patterns.
- [ ] CHK003 The dashboard feels like a developer productivity cockpit, not only a static grid of cards.
- [ ] CHK004 Search, recent history, favorites, workflow groups, and capability entry points have clear hierarchy and can be scanned quickly.
- [ ] CHK005 The dashboard allows users to reach core capabilities in at most two clicks.
- [ ] CHK006 Empty states, loading states, success states, error states, hover states, focus states, and copy/favorite feedback are defined for the main screens.
- [ ] CHK007 Motion is purposeful, fast, and used for state changes, navigation continuity, copy/favorite feedback, loading, and capability transitions.
- [ ] CHK008 Reduced-motion behavior is supported and does not break usability.
- [ ] CHK009 Interactive controls have visible focus states and consistent spacing.
- [ ] CHK010 Theme switching does not break layout, readability, contrast, or motion behavior.
