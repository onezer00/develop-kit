# SAAS-DEV

Developer Productivity Platform planned with GitHub Spec Kit and implemented with Codex.

## Current State

This repository currently contains product discovery, constitution, feature specification, implementation plan, and task breakdown. Application code has not been scaffolded yet.

Codex is the active Spec Kit integration.

## Structure

```text
.agents/skills/      Codex skills generated by Spec Kit
.specify/            Product discovery, constitution, templates, scripts, and integration state
specs/               Feature specs, plans, tasks, and checklists
```

## Spec Kit Workflow With Codex

Use the generated Codex skills:

- `$speckit-specify` for feature specification
- `$speckit-clarify` for resolving ambiguous requirements
- `$speckit-plan` for implementation planning
- `$speckit-tasks` for task generation
- `$speckit-implement` for executing approved tasks
- `$speckit-checklist`, `$speckit-analyze`, and `$speckit-converge` for quality gates

The active feature is:

```text
specs/001-mvp-developer-productivity-platform/
```

## Implementation Rule

Do not implement application code until the active spec, plan, tasks, and checklists are consistent. The product documents under `.specify/product` and the constitution under `.specify/memory/constitution.md` remain the source of truth.
